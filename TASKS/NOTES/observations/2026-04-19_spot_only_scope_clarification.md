Сергей подтвердил: **CIS V10.0-r1 строится исключительно для спот-трейдинга**. Никаких фьючерсов, маржи, опционов. Только покупка и продажа монет на споте Binance.

Это уточнение влияет на:
1. Приоритет механизмов уточнения прогнозов (roadmap из предыдущей observation)
2. Выбор таймфреймов (какие реально нужны, какие нет)
3. Профили Task Awareness в Context Orchestrator (Task 5)
4. Требования к uncertainty bands и calibration

---

## Почему это важно

Spot и фьючерсы — **разные задачи прогнозирования**:

**Спот:**
- Таймфрейм решений: часы-дни
- Критично: directional accuracy + calibration
- Некритично: точная цена входа (±0.5% не определяет успех)
- Нет liquidation risk → uncertainty bands важны для trust, не для выживания

**Фьючерсы с плечом:**
- Таймфрейм решений: минуты-часы
- Критично: точность цены + uncertainty bands + funding rate
- Плечо × волатильность = риск ликвидации

CIS V10.0-r1 строится для спота → требования **мягче**, но фокус другой.

## Следствия для архитектуры

### Таймфреймы

**Реально нужны для спота:**
- `1h` — короткий сигнал, Fast Lane детерминированный
- `4h` — средний горизонт, Fast Lane + Slow Lane async
- `24h` — основной для спот-решений, Slow Lane с полным KB

**Не нужны для спота:**
- `1m`, `5m`, `15m` — спот-трейдер не принимает решения каждые минуты
- Высокочастотные сигналы не имеют практической ценности при спот-комиссиях

**Это упрощает MDC:** три таймфрейма вместо шести-семи.

### Task Awareness профили (для TASK_05)

Достаточно 4 профилей:
- `forecast_1h` — Fast Lane
- `forecast_4h` — Fast Lane с Slow Lane post-forecast
- `forecast_24h` — Slow Lane с полным KB context
- `research` / `postmortem` — без real-time требований

Не нужно добавлять профили `scalping_1m`, `hft_signal` и т.п.

### Приоритет roadmap механизмов уточнения (пересмотр)

Из 8 механизмов roadmap:

**Топ-приоритет для спота:**
- #3 Multi-timeframe confirmation — идеально подходит для спот-решений на согласии 4h+24h
- #1 Bias correction — ваша идея работает лучше на дневных данных чем на минутных
- #4 Volatility regime calibration — для uncertainty bands
- #5 Pattern confidence inheritance — на дневных таймфреймах паттерны стабильнее

**Средний приоритет:**
- #7 Cross-symbol contagion — для альтов на споте (BTC dominance критично)
- #6 Correlation conflict — больше для explainability

**Низкий приоритет для спота (пересмотр):**
- #2 Orderbook sentiment — теряет актуальность на часовых/дневных решениях
- #8 Session-aware calibration — на спот-решениях часы дня не решают

## Пересмотренная оценка финального качества для спота

С 4-5 ключевыми механизмами (без избыточных):

- Directional accuracy 1h: 55-58% → **60-64%**
- Directional accuracy 4h: 58-62% → **64-68%**
- Directional accuracy 24h: 60-65% → **66-70%**
- MAPE 4h/24h: **15-25%**
- Calibration error: **5-10%**
- Coverage: **70-85%** (система молчит когда не уверена)

**Результат:** реально полезный инструмент для информированных спот-решений. Не автотрейдер, но честный справочник с калиброванной уверенностью.
